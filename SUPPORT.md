# Support

C² does not provide private or guaranteed-response support.

## Use YouTube comments for

- brief reactions to an episode;
- a short clarification;
- feedback that does not require a circuit, files, or several steps.

## Use GitHub Discussions for

- conceptual questions;
- help with a circuit;
- unexpected Lab behaviour;
- interface or simulator problems;
- suggestions related to a published laboratory;
- questions that may help other learners later.

Community: <https://github.com/orgs/c2-foundations/discussions>

## Use repository Issues for

Issues are work items, not the first support channel.

A maintainer should create or link an Issue after a Discussion has established
that the behaviour is reproducible and is probably a software defect.

## Security

Never disclose vulnerabilities, tokens, credentials, or exploit details in a
public Discussion or Issue. Follow [SECURITY.md](SECURITY.md) and use private
vulnerability reporting for the affected public repository.

## Response expectations

Questions may be answered by the creator or by other community members. There
is no guaranteed response time.
